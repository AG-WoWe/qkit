qtlab adaptions to get it to run with iPython 2.7 notebooks (AS, JB 01/2015)


* change qtlab/source/lib/misc.py file to

def get_ipython():
    import IPython
    if ipython_is_newer((0, 11)):
        return IPython.get_ipython()
    else:
        return IPython.ipapi.get()
		
* make sure to have a pyvisa installation < 1.5, it works with pyvisa 1.4 (win32)

* install drivers for usb hub

Fixes for GUI:
* add in 02_qtlab_start.py
  after     qt.flow.start_gui()
    qt.mstart()
    qt.msleep(1)
    qt.mend()

* add in instrument.py in set and get methods qt.msleep() before return result: qt.msleep() updates the GUI

* add qt.msleep(3) [values greater 1s typically needed!] after adding one or more instruments to refresh gui
   -> Insert one at end of 0_create_instruments_UFO.py (startscript)
   
* copy kernel manager files in the directories described in E:\IPython_patches

	for ipython 2.xx.x
	notebookapp.py goes to Python27\Lib\site-packages\IPython\html
	kernelmanager.py goes to Python27\Lib\site-packages\IPython\html\services\kernels
	ipynblogo.png goes to Python27\Lib\site-packages\IPython\html\static\base\images

profile_qtlab
	The profile for the notebook server, needs to be placed in the ipython home directory
	Change your notebook.bat to "ipython notebook  --profile=qtlab" to select this profile

and activate kernel_sharing by setting boolean variable shared_kernel in Python27\Lib\site-packages\IPython\html\services\kernels\kernelmanager.py to True

Matplotlib Fix:
* In C:\Python27\Lib\site-packages\matplotlib\mpl-data\matplotlibrc set
	backend: nbagg (for notebooks)
	or	backend: tk (to use qtlab and matplotlib in console)
*Execute "%matplotlib inline" in your notebooks (not sure if needed)
